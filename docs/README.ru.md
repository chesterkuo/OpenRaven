# OpenRaven

**Платформа управления интеллектуальными активами на базе ИИ — автоматически извлекает, организует и активирует профессиональные знания из ваших документов.**

**Читать на других языках:**
[English](../README.md) | [繁體中文](README.zh-TW.md) | [简体中文](README.zh-CN.md) | [日本語](README.ja.md) | [한국어](README.ko.md) | [Français](README.fr.md) | [Español](README.es.md) | [Nederlands](README.nl.md) | [Italiano](README.it.md) | [Tiếng Việt](README.vi.md) | [ไทย](README.th.md) | **Русский**

OpenRaven преобразует разрозненные документы — PDF, DOCX, презентации, транскрипты совещаний, экспорты из Notion — в структурированный граф знаний с возможностью запросов. Задавайте вопросы на естественном языке, исследуйте связи между концепциями, создавайте статьи вики и разрабатывайте курсы на основе вашей базы знаний.

## Почему OpenRaven?

Специалисты теряют институциональные знания при смене должностей или организаций. Исследования показывают, что 42% институциональных знаний существует только в головах людей (IDC). OpenRaven фиксирует и структурирует эти знания, чтобы они всегда были доступны, поддавались поиску и могли быть переданы другим.

## Возможности

### Движок знаний
- **Умный импорт** — загружайте PDF, DOCX, PPTX, XLSX, Markdown, изображения (компьютерное зрение ИИ) или экспорты из Notion/Obsidian. Сущности и связи извлекаются автоматически.
- **Граф знаний** — интерактивная визуализация графа с силовой раскладкой, фильтрацией по типу сущности, силе связей и поиском. Экспорт в формате GraphML или PNG.
- **Вопросы и ответы на естественном языке** — задавайте вопросы базе знаний, используя 6 режимов запросов (mix, local, global, hybrid, keyword, direct LLM). Ответы содержат ссылки на источники.
- **Автоматически сгенерированная вики** — статьи автоматически создаются на основе извлечённых сущностей и связей.
- **Создание курсов** — разрабатывайте структурированные курсы из базы знаний с планированием учебной программы, генерацией глав и интерактивным экспортом в HTML.
- **Аналитика открытий** — автоматический анализ тематических кластеров, пробелов и тенденций в знаниях.

### Коннекторы
- **Google Drive** — импорт документов (PDF, Docs, Sheets, Slides)
- **Gmail** — импорт писем в виде записей базы знаний
- **Google Meet** — импорт транскриптов совещаний через Drive API
- **Otter.ai** — импорт транскриптов совещаний через API key

### Вертикальные схемы
- **Base** — универсальное извлечение сущностей (по умолчанию)
- **Engineering** — техническая архитектура, системы, API
- **Finance** — компании, финансовые метрики, нормативные акты
- **Legal (Taiwan)** — законодательные акты, судебные решения, правовые принципы (традиционный китайский)
- **Finance (Taiwan)** — компании, котирующиеся на TWSE, финансовые метрики (традиционный китайский)

### Поддержка нескольких языков

OpenRaven поддерживает 12 языков с автоматическим определением языка браузера и ручным переключением:

| Язык | Код | Язык | Код |
|------|-----|------|-----|
| Английский | `en` | Итальянский | `it` |
| Традиционный китайский | `zh-TW` | Вьетнамский | `vi` |
| Упрощённый китайский | `zh-CN` | Тайский | `th` |
| Японский | `ja` | Русский | `ru` |
| Корейский | `ko` | Французский | `fr` |
| Испанский | `es` | Нидерландский | `nl` |

**Принцип работы:**
- Язык браузера/ОС определяется автоматически при первом посещении (резервный вариант: английский)
- Пользователи могут переключаться через выбор языка в навигационной панели
- Предпочтение сохраняется в localStorage (немедленно) и в профиле пользователя (синхронизация между устройствами)
- Ответы LLM соответствуют выбранному языку пользователя
- Статьи вики и содержимое курсов следуют языку исходного документа
- Метки графа знаний остаются на английском языке

### Корпоративные возможности (Managed SaaS)
- **Изоляция нескольких арендаторов** — отдельные базы знаний для каждого арендатора с раздельным хранилищем
- **Аутентификация** — электронная почта/пароль + Google OAuth 2.0 с управлением сессиями
- **Журнал аудита** — отслеживание всех действий пользователей с экспортом в CSV
- **Управление командой** — приглашайте участников в своё рабочее пространство
- **Neo4j Graph Backend** — хранилище графов производственного уровня (опционально, по умолчанию: NetworkX)
- **Развёртывание Docker Compose** — развёртывание одной командой с nginx, PostgreSQL, Neo4j

## Архитектура

```
openraven/                  # Python backend (FastAPI + LightRAG + LangExtract)
  src/openraven/
    api/server.py           # FastAPI app factory, все API endpoint'ы
    pipeline.py             # Основной pipeline: импорт, запросы, граф, вики, курсы
    graph/rag.py            # LightRAG wrapper с запросами, учитывающими язык
    auth/                   # Система аутентификации (сессии, OAuth, сброс пароля)
    audit/                  # Модуль журналирования аудита
  alembic/                  # Миграции базы данных
  tests/                    # 159+ тестов на Python

openraven-ui/               # TypeScript frontend (React 19 + Vite 6 + Tailwind 4)
  src/
    i18n.ts                 # Инициализация i18next (12 языков, 11 пространств имён)
    App.tsx                 # Корневой компонент с маршрутами + навигационной панелью
    pages/                  # 14 компонентов страниц
    components/             # LanguageSelector, GraphViewer, ChatMessage и др.
    hooks/useAuth.tsx       # Auth context с синхронизацией языка
  public/locales/           # 132 файла JSON переводов (12 языков x 11 пространств имён)
  server/index.ts           # Hono BFF (API proxy + обслуживание статических файлов)
  tests/                    # 46 тестов Bun

ecosystem.config.cjs        # Конфигурация развёртывания PM2
```

## Быстрый старт

### Предварительные требования
- Python 3.12+
- Bun 1.0+
- Node.js 20+ (для PM2)

### 1. Клонирование и установка

```bash
git clone https://github.com/nickhealthy/OpenRaven.git
cd OpenRaven

# Backend
cd openraven
python3 -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"

# Frontend
cd ../openraven-ui
bun install
```

### 2. Настройка

```bash
# openraven/.env
GEMINI_API_KEY=your-gemini-api-key     # Обязательно: провайдер LLM
WORKING_DIR=/path/to/knowledge-data     # Где хранятся данные базы знаний

# Опционально: включить функции managed SaaS
# DATABASE_URL=postgresql://user:pass@localhost:5433/openraven
# GOOGLE_CLIENT_ID=...
# GOOGLE_CLIENT_SECRET=...
# GRAPH_BACKEND=neo4j
# NEO4J_URI=bolt://localhost:7687
```

### 3. Запуск через PM2

```bash
# Из корневой директории проекта
pm2 start ecosystem.config.cjs

# Проверить статус
pm2 status

# Просмотреть журналы
pm2 logs
```

Сервисы:
- **openraven-core** (порт 8741) — сервер Python API
- **openraven-ui** (порт 3002) — BFF + frontend

### 4. Сборка frontend для продакшена

```bash
cd openraven-ui
bun run build          # Сборка в dist/
pm2 restart openraven-ui
```

Откройте http://localhost:3002 в браузере.

### Альтернатива: Docker Compose

```bash
docker compose up -d
```

Запускает nginx (порт 80), PostgreSQL, Neo4j, сервер API и сервер UI.

## Разработка

```bash

cd openraven && python3 -m pytest tests/ -v

cd openraven-ui && bun test tests/

# Бенчмарки (требуется GEMINI_API_KEY)

```

### Добавление переводов

Файлы переводов находятся в `openraven-ui/public/locales/{locale}/{namespace}.json`.

Чтобы добавить или обновить перевод:
1. Отредактируйте JSON-файл для целевого языка
2. Ключи должны совпадать с ключами в англоязычном исходном файле
3. Сохраняйте placeholder'ы `{{interpolation}}`
4. Выполните `bun run build` и перезапустите PM2

Чтобы добавить новый язык:
1. Создайте новую директорию в `public/locales/` (например, `de/`)
2. Скопируйте все JSON-файлы из `en/` и переведите значения
3. Добавьте код языка в `SUPPORTED_LNGS` в `src/i18n.ts`
4. Добавьте язык в массив `LOCALES` в `src/components/LanguageSelector.tsx`
5. Добавьте язык в `SUPPORTED_LOCALES` в `openraven/src/openraven/auth/routes.py`
6. Добавьте название языка в `LOCALE_NAMES` в `openraven/src/openraven/graph/rag.py`

## Обзор API

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `POST` | `/api/ask` | Запрос к базе знаний (поддерживает параметр locale) |
| `POST` | `/api/ingest` | Загрузка и обработка документов |
| `GET` | `/api/graph` | Получение данных графа знаний |
| `GET` | `/api/wiki` | Список статей вики |
| `GET` | `/api/status` | Статистика базы знаний |
| `GET` | `/api/discovery` | Автоматически сгенерированная аналитика |
| `POST` | `/api/courses/generate` | Создание курса |
| `GET` | `/api/connectors/status` | Статус коннекторов |
| `PATCH` | `/api/auth/locale` | Обновление языковых предпочтений пользователя |
| `GET` | `/api/audit` | Журнал аудита (с постраничной разбивкой) |

Полная документация API доступна по адресу http://localhost:8741/docs (автоматически генерируется FastAPI).
